## 🛠️ Proyectos Destacados

Aquí detallo la arquitectura y los problemas reales que resuelven mis desarrollos principales. (El código fuente de proyectos comerciales se mantiene privado por motivos de seguridad y propiedad intelectual, pero puedes ver demos funcionales en mi portfolio).

### 🍉 1. SandiaCatalog (SaaS Multi-tenant)
Una plataforma escalable diseñada para que pequeños comercios digitalicen sus ventas vía WhatsApp sin necesidad de conocimientos técnicos.

🚀 **[Ver Catálogo de Muestra / Contratar Servicios](https://sandiacatalog.web.app/catalogo/?uid=vufWNXyePsVtiYNcRYXA0zvnC9l1)**

* **El Problema:** Las plataformas de e-commerce tradicionales son complejas y costosas para negocios locales que solo necesitan un catálogo rápido y autogestionable.
* **La Solución Técnica:** Desarrollé una arquitectura *multi-tenant* donde cada cliente administra su propio espacio. 
* **Stack:** `Next.js 16`, `React`, `TypeScript`, `Tailwind CSS`, `Firebase`.
* **Highlights Arquitectónicos:**
    * Implementación de aislamiento de datos mediante reglas estrictas en **Firebase Auth y Firestore**.
    * Sistema de compresión de imágenes a **Base64** en el lado del cliente antes de subir a la base de datos, optimizando drásticamente los costos de almacenamiento Cloud.
    * Generación dinámica de códigos QR y URLs únicas para distribución instantánea de los catálogos.

### 🤖 2. Proyecto_Facturas (Bot de Automatización)
Script de automatización diseñado para eliminar el trabajo manual repetitivo en entornos corporativos.
* **El Problema:** Un proceso ineficiente obligaba a un operador a extraer manualmente cientos de líneas de datos de consumo desde facturas telefónicas en PDF para cargarlas en Excel, perdiendo horas de trabajo.
* **La Solución Técnica:** Aprovechando mi experiencia en lenguajes de scripting, desarrollé un bot que procesa archivos masivos de forma autónoma.
* **Stack:** `Python`, librerías de procesamiento de PDF y manipulación de datos.
* **Highlights Arquitectónicos:**
    * Extracción estructurada de información clave (Número de línea y Consumo) descartando el "ruido" del documento.
    * Generación automática de un reporte final estandarizado en formato Excel.
    * **Impacto:** Reducción de un proceso manual de varias horas a una ejecución automatizada de **menos de 2 minutos**.

### 📊 3. Cuentas Claras (Gestor Financiero Colaborativo)
Aplicación web diseñada para la previsión del flujo de caja en economías domésticas compartidas.
* **El Problema:** La gestión financiera entre múltiples miembros de un hogar suele ser desordenada, dificultando saber cuánto dinero real hay disponible para cubrir los gastos fijos.
* **La Solución Técnica:** Creé una interfaz dinámica que calcula y renderiza en tiempo real el balance general del mes.
* **Stack:** `React`, `Next.js`.
* **Highlights Arquitectónicos:**
    * Control colaborativo de ingresos y egresos.
    * Lógica de renderizado en tiempo real que compara el dinero disponible actual versus los gastos fijos proyectados.
    * Optimización del proyecto para un despliegue rápido y a costo cero en la nube.
